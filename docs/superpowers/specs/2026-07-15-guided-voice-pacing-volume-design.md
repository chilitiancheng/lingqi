# 冥想引导人声节奏与音量设计

## 目标

让六类冥想中的中文引导人声更柔和、更从容，同时保持环境音、呼吸滴嗒提示音和系统媒体音量逻辑不变。

## 已确认体验参数

- 语速从 `0.82` 调整为 `0.72`。
- 单次 TTS 播放音量调整为媒体通道的 `65%`。
- 音调继续使用 `0.92`，避免同时改变声音性格。
- 只影响 `GuidedVoiceController` 播放的人声；环境音与 `CuePlayer` 不调整。

## 实现设计

新增一个纯 Kotlin 的 `GuidedVoiceProfile`，集中保存语速、音调和人声音量，并提供已确认的默认配置。控制器初始化 TTS 时读取语速和音调；每次播报时通过 `TextToSpeech.Engine.KEY_PARAM_VOLUME` 的 `Bundle` 传入人声音量。

配置对象约束三项参数必须大于零，其中音量不得超过 `1.0`，防止后续误配导致静音、异常语速或音量溢出。单元测试验证默认值和参数边界；Android 构建验证 TTS 接口调用可编译。

## 验收

- 默认引导人声参数为 `0.72 / 0.92 / 0.65`。
- 非法参数在创建配置时立即失败。
- 冥想引导状态测试与完整 JVM 单元测试通过。
- Debug APK 可构建；真机听感留给安装后的人工验收。

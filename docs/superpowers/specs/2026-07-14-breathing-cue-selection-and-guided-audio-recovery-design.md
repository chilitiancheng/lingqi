# 灵栖呼吸提示音选择与冥想声音恢复设计

## 背景与已确认问题

478 呼吸提示音目前只有实时合成的“滴/嗒”，真机能够播放，但用户认为音量仍偏小，并希望默认改为真实老式摆钟的机械咔哒声，同时允许在个人中心自主选择提示音。

其余五类冥想当前应播放中文语音引导和离线背景音乐，但真机验收为两者均无可感知声音。代码审查确认：

- 背景音乐虽已正确打包进 APK，但播放器又将左右声道增益固定为 `0.10f`，手机媒体音量之外还被 App 额外衰减到 10%。
- App 以 Android 16 为目标版本，但清单的 `<queries>` 中没有 Android 11 起 TTS 所需的 `android.intent.action.TTS_SERVICE` 查询声明。
- TTS 控制器忽略初始化、中文语言支持和 `speak()` 的失败结果，界面也没有失败状态，因此任何 TTS 故障都会表现为静默无声。

中文引导词文件本身是有效 UTF-8，背景音乐也存在于已构建 APK，因此本轮不替换引导文案或环境音乐素材。

## 目标

- 在个人中心提供“摆钟咔哒 / 清脆短铃 / 静音”三种呼吸提示音选择，默认使用真实摆钟录音。
- 保持 478 的 4 秒收缩、7 秒停留、8 秒散开及原有触发时点。
- 恢复身体扫描、睡前放松、专注、情绪舒缓、晨间唤醒五类冥想的中文语音引导和离线背景音乐。
- 所有声音继续使用 Android 媒体通道，由手机媒体音量键控制最终音量。
- 声音加载或 TTS 不可用时不崩溃、不阻断计时，并向用户提供可理解的状态。

## 非目标

- 不新增 App 内音量滑杆、均衡器或每种冥想独立音量。
- 不让摆钟连续走时；每个阶段入口只播放一次短咔哒。
- 不改变睡眠检测、麦克风采集、提醒、导出、Provider 或数据库加密逻辑。
- 不更改星群造型、呼吸动画或 4/7/8 计时。
- 不把原始 Pixabay 长录音作为可独立访问、分享或导出的文件提供给用户。
- 不增加云端语音、网络播放或运行时素材下载。

## 用户设置与兼容

新增 `BreathingCueSound` 枚举，持久化值固定为：

| 值 | 用户文案 | 行为 |
| --- | --- | --- |
| `PENDULUM` | 摆钟咔哒 | 播放真实录音裁出的 tick / tock，默认值 |
| `BELL` | 清脆短铃 | 播放当前实时合成的滴 / 嗒 |
| `SILENT` | 静音 | 不播放 478 阶段提示音 |

`UserPreferences` 新增 `breathingCueSound`。读取旧版 JSON 时字段缺失、为空或出现未知值，一律安全回退到 `PENDULUM`；保存时写入稳定英文枚举值。该变化不修改数据库表结构，只扩展现有偏好 JSON。

现有 `soundEnabled` 保留，用途收窄为“语音与环境声”，只控制非 478 冥想的 TTS 与背景音乐。它不覆盖用户选择的呼吸提示音。

播放器右上角扬声器按钮仍是当前练习的临时总静音：

- 关闭时，本次练习的摆钟、短铃、TTS 和背景音乐全部停止。
- 再次打开后，只恢复当前允许的声音，不补播已经错过的呼吸提示或旧引导句。
- 临时总静音不写回个人中心偏好。

最终播放条件为：

- 478 提示音：会话未暂停、未弹出退出确认、未完成、临时总静音未开启，且 `breathingCueSound != SILENT`。
- 其他五类语音与背景音：上述会话条件成立、临时总静音未开启，且 `soundEnabled == true`。

## 个人中心交互

“提醒与声音”区按以下顺序显示：

1. “呼吸提示音”，副标题显示当前选择；整行可点击。
2. “语音与环境声”，保留开关，副标题为“用于引导冥想”。
3. 原有冥想提醒与睡眠提醒。

点击“呼吸提示音”打开与现有黑白极简界面一致的单选对话框。三项均显示名称，当前项使用白色选中标记；点击任一项立即保存并关闭。选择“静音”只影响 478 阶段提示，不影响中文引导或背景音乐。

## 摆钟素材与离线处理

采用用户确认的 Pixabay 官方素材：

- 标题：Old Clock Ticking
- 作者：Universfield
- 官方页面：https://pixabay.com/sound-effects/film-special-effects-old-clock-ticking-352288/
- 页面标注时长：14 秒
- 发布日期：2025-06-02
- 许可：Pixabay Content License
- 许可摘要：https://pixabay.com/service/license-summary/

实施时只从官方素材页取得原文件。原文件保留在临时处理目录，不进入 Git 和 APK。通过波形与实听选择一组自然相邻的 tick / tock，分别裁成不超过 300 ms 的单声道 44.1 kHz PCM WAV：去除无关前后静音，使用极短淡入淡出避免点击爆音，移除直流偏移并做峰值归一化，最高峰不超过 -1 dBFS，不加入合成谐波或混响。

派生的两个短片段作为 App 内部资源打包，文件名稳定为 `pendulum_tick.wav` 与 `pendulum_tock.wav`。`assets/audio/licenses.json` 增加原素材标题、作者、官方 URL、发布日期、下载日期、许可 URL、原文件 SHA-256、两个派生文件 SHA-256、裁切时间点和处理参数。现有环境音乐许可记录保留不动。

## 478 播放设计

短录音使用面向低延迟短音效的播放器预加载；播放器创建时使用 `USAGE_MEDIA` 和适合提示音的内容类型。进入 478 页面即开始加载，加载成功后：

- 收缩开始播放 `pendulum_tick.wav` 一次。
- 散开开始播放 `pendulum_tock.wav` 一次。
- 停留阶段不播放。
- 同一阶段重组或页面重绘不得重复触发。

真实录音按资源原幅度播放，最终响度由手机媒体音量控制。若所选 `PENDULUM` 资源未能加载，对当前会话自动回退到 `BELL`；若合成短铃也播放失败，则保持静音但不影响计时。`SILENT` 不创建或触发音轨。

暂停、退出确认、页面销毁、练习完成或临时总静音时立即停止短音。恢复练习后等待下一个合法阶段边界，不补播暂停期间错过的 tick / tock。

## 语音引导恢复

AndroidManifest 的既有 `<queries>` 保留灵恋包名，并追加 TTS 服务 intent 查询。TTS 控制器明确区分 `INITIALIZING`、`READY`、`UNAVAILABLE` 三种状态：

- 只有初始化成功且简体中文返回可用时进入 `READY`。
- 首句在初始化期间只保留一条待播内容；进入 `READY` 后立即播放，避免页面刚打开时丢失。
- 初始化失败、中文缺失或 `speak()` 返回失败时进入或上报 `UNAVAILABLE`，清除待播内容。
- 使用 utterance progress listener 记录开始、完成与错误，不能把“调用了 speak”误判为已经播放。
- 暂停或临时静音时停止当前 utterance，但不推进尚未到时间的后续引导。
- 恢复后不重播已经完整播放的句子；被暂停打断的句子也不自动从头补播，等待下一条时间点。

页面在 `UNAVAILABLE` 时显示一次非阻塞提示：“中文语音暂不可用，请检查系统文字转语音设置”。提示不得覆盖计时和退出操作，也不得自动跳转系统设置。用户仍可只听背景音乐完成练习。

## 背景音乐恢复

继续使用已打包的 `meditation-relax-sleep-music-346733.mp3`，仅用于五类非 478 引导冥想。播放器保持循环、`USAGE_MEDIA` 和本地同步准备方式。

移除额外的 10% 固定衰减，播放器资源增益设为完整基准；最终音量只由手机媒体音量控制。暂停、退出确认、完成、页面销毁、个人中心“语音与环境声”关闭或本次练习临时总静音时，背景音乐停止或暂停；恢复时从原播放位置继续，不重新从头开始。

素材打开、准备或播放失败时，播放器上报可测试的失败状态并安全释放资源。TTS 仍可独立工作；背景音乐失败不能连带关闭语音。

## 状态流与职责

1. `LingqiRepository` 负责兼容读取和保存 `breathingCueSound`。
2. `ProfileScreen` 只负责显示、选择并持久化设置。
3. `MeditationPlayerScreen` 在进入时读取偏好，结合练习类型、临时总静音和暂停状态计算实际播放策略。
4. 呼吸阶段控制器只发出阶段入口事件；音效路由器根据选择调用录音播放器、现有合成播放器或静音实现。
5. `GuidedVoiceController` 独立管理 TTS 生命周期和可用性；`AmbientAudioPlayer` 独立管理背景音乐。任一组件失败不得使另一组件静音。
6. 页面销毁时统一释放短音、TTS 和背景音乐资源。

## 测试策略

### 单元测试

- 旧偏好 JSON、缺失字段和未知枚举值默认得到 `PENDULUM`。
- 三种选择均可保存并原样读回，且不改变 `soundEnabled`。
- 呼吸音效路由在 `PENDULUM`、`BELL`、`SILENT` 下分别选择录音、合成和无播放。
- 暂停、退出确认、完成及临时总静音都阻止新提示；恢复后不补播旧阶段。
- `soundEnabled` 只影响非 478 的语音与背景，不影响非静音呼吸提示。
- TTS 初始化期间的首句只排队一次；成功后播放，失败或语言不支持时报告不可用。
- TTS 暂停/恢复和后续 cue 索引不会重复或跳过尚未到达的引导。
- 背景音乐策略仅允许五类非 478 冥想在有效会话中播放。

### 构建与静态验证

- Manifest 合并结果包含 `android.intent.action.TTS_SERVICE` 查询，同时保留灵恋包查询和 signature Provider 配置。
- APK 包含两个派生摆钟 WAV、现有环境音乐和更新后的许可清单。
- 校验两个 WAV 为单声道 44.1 kHz、时长不超过 300 ms、峰值不削波、首尾接近零且非静音。
- 校验派生资源与许可清单 SHA-256 一致。
- 运行单元测试、Debug 构建、AndroidTest 构建、Lint、Release 构建和 `git diff --check`。

### 真机验收

- 个人中心三种呼吸提示音可切换，退出重进后保持选择。
- 默认摆钟在收缩开始响一次 tick、散开开始响一次 tock；机械感自然、清楚，无持续走钟和爆音。
- 清脆短铃仍可选择；静音仅关闭 478 提示音。
- 五类冥想逐一抽验：进入后首句能播放，背景音乐可听见，手机媒体音量键能同步调大、调小和静音。
- 暂停、继续、临时总静音、退出确认和返回页面均无重复首句、补播旧提示或音频泄漏。
- 关闭“语音与环境声”后五类冥想无 TTS 和背景音乐，但 478 仍按其独立选择播放。
- 若设备中文 TTS 确实不可用，页面出现明确提示，背景音乐仍能播放。

## 交付边界

本轮实现仅包含呼吸提示音选择、摆钟派生资源及许可记录、五类冥想 TTS/背景音恢复和对应测试。真机验收前先完成全部电脑端测试与签名构建；真机测试需再次明确占用手机，完成后明确释放。任何新的音量混音、更多音效、在线下载或逐冥想声音方案另行设计，不在本轮顺带扩展。
